What is a Lambda Expression?
📘 Theory:

A lambda expression is a short way to write an implementation of a functional interface (without creating a class or anonymous class).

👉 Introduced in Java 8 to support functional programming

🔹 Basic Syntax
(parameters) -> expression

or

(parameters) -> {
    // multiple statements
}

🔹 Simple Examples
✅ No parameter
() -> System.out.println("Hello")
✅ One parameter
x -> x * 2
✅ Multiple parameters
(a, b) -> a + b

🔹 Key Characteristics (Theory)

No need to write class or method

Works only with functional interfaces

Supports type inference (no need to declare types)

Makes code short and readable

Enables functional programming style

🔹 Lambda vs Anonymous Class
Feature	Lambda	Anonymous Class
Code size	Short	लंबा (long)
Syntax	Simple	Complex
Readability	High	Low
Use	Functional interface	Any interface/class

🔹 Real-Time Usage

✅ 1. With Runnable (Multithreading)
Runnable r = () -> System.out.println("Thread running");
new Thread(r).start();

✅ 2. With Collections (forEach)
list.forEach(x -> System.out.println(x));

✅ 3. With Streams
list.stream()
    .filter(x -> x > 5)
    .map(x -> x * 2)
    .forEach(System.out::println);


🔹 Advantages

✔ Reduces boilerplate code

✔ Improves readability

✔ Encourages functional style

✔ Works well with Streams API

✔ Easy parallel processing


🔹 Limitations

❌ Only works with functional interfaces

❌ Can be confusing for beginners

❌ Not suitable for complex logic (better use methods)


🔹 Important Points (Exam)

Lambda = implementation of functional interface

Uses arrow operator (->)

Can replace anonymous classes

Cannot be used without functional interface


🔹 Example (Full)
@FunctionalInterface
interface Add {
    int sum(int a, int b);
}

Add obj = (a, b) -> a + b;
System.out.println(obj.sum(5, 3));











1. Method Reference ( :: )
📘 Theory:

A method reference is a shorthand way of writing a lambda expression when the lambda just calls an existing method.

👉 Instead of:

x -> System.out.println(x)

👉 You write:

System.out::println
🔸 Types of Method References
✅ 1. Static Method Reference
ClassName::staticMethod
Example:
Function<Integer, Integer> square = x -> Math.abs(x);

➡ Using method reference:

Function<Integer, Integer> square = Math::abs;
✅ 2. Instance Method Reference
object::methodName
Example:
Consumer<String> c = x -> System.out.println(x);

➡ Using method reference:

Consumer<String> c = System.out::println;
✅ 3. Constructor Reference
ClassName::new
Example:
Supplier<ArrayList<String>> list = () -> new ArrayList<>();

➡ Using method reference:

Supplier<ArrayList<String>> list = ArrayList::new;
✅ 4. Instance Method of Arbitrary Object
ClassName::methodName
Example:
list.sort((a, b) -> a.compareTo(b));

➡ Using method reference:

list.sort(String::compareTo);
🔹 2. Chaining (Very Important)

Chaining means combining multiple functional operations together

🔸 Function Chaining
Methods:

andThen() → executes next

compose() → executes before

✅ Example:
Function<Integer, Integer> multiply = x -> x * 2;
Function<Integer, Integer> add = x -> x + 3;

// andThen
System.out.println(multiply.andThen(add).apply(5)); // (5*2)+3 = 13

// compose
System.out.println(multiply.compose(add).apply(5)); // (5+3)*2 = 16
🔸 Predicate Chaining
Methods:

and()

or()

negate()

✅ Example:
Predicate<Integer> p1 = x -> x > 5;
Predicate<Integer> p2 = x -> x < 10;

System.out.println(p1.and(p2).test(7)); // true
System.out.println(p1.or(p2).test(3));  // true
System.out.println(p1.negate().test(3)); // true
🔸 Consumer Chaining
Method:

andThen()

✅ Example:
Consumer<String> c1 = x -> System.out.println(x);
Consumer<String> c2 = x -> System.out.println(x.toUpperCase());

c1.andThen(c2).accept("java");
🔹 3. Why These Are Important

👉 Used heavily in:

Streams API

Data pipelines

Clean, modular code

Functional-style programming

🔹 Quick Summary 🧠
Method Reference:

👉 Shortcut for lambda
👉 Uses ::

Chaining:

👉 Combine multiple operations

🔹 Interview Tip ⭐

If asked:

“Difference between andThen() and compose()?”

👉 Answer:

andThen() → current function runs first

compose() → parameter function runs first

✅ Final One-Line

👉 Method reference simplifies lambda, chaining combines multiple operation
